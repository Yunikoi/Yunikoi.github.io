🚀 Sentio 2.0：（6个月实战路线图）

📅 阶段总览

阶段 时间 核心目标 关键产出
Phase 1 第 1–4 周 编程与 App 基础 基础 App UI 框架
Phase 2 第 5–8 周 后端与数据库 打通“前端-后端-数据库”
Phase 3 第 9–12 周 IoT 数据采集 个人行为原始数据库
Phase 4 第 13–16 周 AI 行为识别 行为识别模型接口
Phase 5 第 17–20 周 全系统集成 Sentio 2.0 完整原型
Phase 6 第 21–24 周 MEXT 与教授套磁 研究计划书 + 演示视频

🟢 Phase 1：编程地基 (Week 1–4)

专注于将你的 React Native 技能与基础逻辑结合。

  • Week 1: Python/Java 基础语法（变量、条件判断、循环）。
  • Week 2: 面向对象编程（类与对象、继承）——任务： 写一个用户信息管理脚本。
  • Week 3: React Native 深度实践（组件通信、路由导航）。
  • Week 4: 静态 App 开发——任务： 完成 Sentio 的主界面、数据监测页、个人中心页。

🟡 Phase 2：后端联通 (Week 5–8)

让你的 App 拥有“大脑”和“记忆”。

  • Week 5: 网络协议基础（HTTP 状态码、JSON 格式、RESTful API 概念）。
  • Week 6: Flask (Python) 或 Spring Boot (Java) 快速入门——任务： 写出第一个“Hello World”接口。
  • Week 7: 数据库基础（MySQL/SQLite）——任务： 设计 User 表和 SensorData 表。
  • Week 8: 集成任务： 实现 App 登录功能，并将 App 上的点击行为记录到服务器数据库中。

🟠 Phase 3：IoT 数据流 (Week 9–12)

这是你作为“IoT 选手”的核心竞争力：处理真实世界的数据。

  • Week 9: 传感器原理（加速度计 Accelerometer、陀螺仪 Gyroscope）。
  • Week 10: 数据采集——任务： 编写 App 脚本，以 20Hz-50Hz 的频率采集行走、坐下、摔倒的传感器原始数据。
  • Week 11: 数据清洗（去噪、时间戳对齐）。
  • Week 12: 集成任务： 实现“实时上传”功能，将手机传感器数据流式传输到后端。

🔵 Phase 4：AI 算法建模 (Week 13–16)

利用 AI 赋予系统智慧。

  • Week 13: 机器学习基础（特征提取：均值、方差、过零率）。
  • Week 14: 使用 scikit-learn 训练分类模型（如随机森林或 SVM）。
  • Week 15: 模型评估（准确率、召回率）与调优。
  • Week 16: 模型部署： 将训练好的模型封装成 API，后端接收到传感器数据后，自动返回“当前正在走路”或“疑似摔倒”。

🟣 Phase 5：系统整合与优化 (Week 17–20)

将所有碎片拼成一个完整的“作品”。

  • Week 17: 串联全链路：传感器 → App → 后端 → AI 模型 → App 结果展示。
  • Week 18: 异常处理（网络断开怎么办？数据丢包怎么办？）。
  • Week 19: UI/UX 优化（加入数据可视化图表，如折线图展示活动量）。
  • Week 20: 定稿： 录制一个 3 分钟的 Demo 演示视频（这是套磁的神器）。

🔴 Phase 6：学术包装与套磁 (Week 21–24)

  • Week 21: 撰写研究计划书： 将 Sentio 2.0 提升到学术高度（讨论隐私保护、边缘计算等）。
  • Week 22: 目标实验室筛选（寻找 HCI、Ubiquitous Computing 方向的教授）。
  • Week 23: 模拟面试准备（准备好回答：为什么用这个模型？你的系统架构创新在哪里？）。
  • Week 24: 发送套磁邮件（带上你的 GitHub 链接和 Demo 视频）。
